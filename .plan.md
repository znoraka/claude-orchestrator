# Plan: Shell Process Running Indicator in Sidebar

## Goal
Show in the sidebar whether a command is currently running in any shell tab for each workspace/worktree.

## Changes

### 1. Rust: Store child PID in PtySession (`pty_manager.rs`)
- Add `child_pid: Option<u32>` to `PtySession`
- Capture PID from `spawn_command().process_id()` in both `create_session` and `create_shell_session`
- Add method `has_running_child(&self, session_id: &str) -> Result<bool, String>` that runs `pgrep -P <pid>` to check if the shell has any child processes

### 2. Rust: Add Tauri command (`lib.rs`)
- Add `pty_has_running_child(session_id: String) -> Result<bool, String>` command
- Register it in the invoke handler

### 3. Frontend: Add polling hook (`src/hooks/useShellProcessStatus.ts`)
- New hook that takes a `Map<string, ShellTab[]>` (directory → shell tabs)
- Polls `pty_has_running_child` every 2 seconds for each active shell session
- Returns `Map<string, boolean>` (directory → has running process)

### 4. Frontend: Wire into App.tsx
- Use the hook with `shellTabs` state
- Pass the resulting map down to `Sidebar` as a new prop `shellProcessRunning`

### 5. Frontend: Display indicator in Sidebar
- On each worktree row, if that worktree's directory has a running shell process, show a small terminal/command icon indicator (e.g., a `>_` icon with a pulse or distinct color)
- Placed next to the session count badge on the worktree header row

## Files Modified
- `src-tauri/src/pty_manager.rs` — store PID, add child process check
- `src-tauri/src/lib.rs` — new Tauri command
- `src/hooks/useShellProcessStatus.ts` — new polling hook
- `src/App.tsx` — wire hook, pass prop to Sidebar
- `src/components/Sidebar.tsx` — accept prop, render indicator
